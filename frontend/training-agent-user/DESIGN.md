# training-agent-user 设计说明

## 1. 分层设计
- `src/views`：页面编排与交互
- `src/components`：可复用 UI 组件（含登录/注册弹窗）
- `src/api`：后端接口适配层
- `src/utils`：工具层（请求封装、存储、安全清洗、日志）
- `src/modules`：按业务子域拆分（如在线考试）

## 2. 认证与状态
- 通过 token + 本地存储维持登录态。
- 未登录访问受限能力时，由统一弹窗引导登录。
- 用户身份信息由本地缓存与接口回源协同更新。

## 3. 安全边界
- HTML 渲染统一走 sanitize 边界（`sanitizeHtml` / `escapeHtml`）。
- 日志统一走 `logger`，并对敏感字段进行过滤。
- 生产环境可禁用控制台普通输出，降低敏感信息泄露面。

## 4. 可维护性策略
- API 层与视图层分离，降低页面耦合。
- 复杂场景（如考试流程）放入 `modules/` 独立演进。
- 统一 lint 与 test 命令做提交前校验。
