# 权限变更后会话刷新运维规程

## 1. 适用范围

- 后台管理员进行以下操作后：
  - 角色分配/取消分配用户
  - 用户角色授权变更
  - 用户状态变更
  - 用户密码重置
- 目标系统：`ruoyi-admin` + `RuoYi-Vue3`

## 2. 系统行为说明

- 权限相关变更提交后，后端会强制失效目标用户已登录会话（token）。
- 目标用户必须重新登录，新的角色和权限才会生效。
- 前端会提示：`权限已更新，相关用户需重新登录后生效`。

## 3. 操作流程

1. 管理员完成角色或用户权限调整并保存。
2. 在操作成功提示后，确认页面出现“需重新登录后生效”的提示。
3. 通知受影响用户重新登录。
4. 用户重新登录后，验证菜单、按钮和接口访问是否与新权限一致。

## 4. 验证清单

- 未授权前访问受控接口返回 `403`。
- 授权后旧会话访问受控接口返回未登录或无权限。
- 重新登录后，受控接口返回成功（`200`）。
- 取消授权后，旧会话失效，重新登录后受控接口再次返回 `403`。

## 5. 故障排查

- 现象：用户声称“已授权但仍无权限”
  - 检查是否重新登录。
  - 检查角色权限字符是否包含目标接口权限（如 `train:exam:list/query/add/edit/remove/export`）。
  - 检查角色菜单是否包含对应页面路由。
- 现象：权限收回后仍可访问
  - 核验是否命中了 `/admin/*` 受控接口。
  - 核验接口是否存在 `@PreAuthorize`。
