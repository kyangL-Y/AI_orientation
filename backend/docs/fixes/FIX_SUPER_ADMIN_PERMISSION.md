# 用户学习时长 - 超级管理员权限修复

## 🐛 问题描述

**问题**: 超级管理员只能看到自己，无法看到所有集团的用户数据

**原因**: 之前的修复对所有用户都添加了 `tenant_id` 过滤，包括超级管理员

**超级管理员的特征**: `tenant_id = '000000'`

---

## ✅ 修复逻辑

### 权限规则
1. **超级管理员** (`tenant_id = '000000'`) → 可以看到所有集团的数据
2. **集团管理员** (`tenant_id != '000000'`) → 只能看到自己集团的数据

### 修复代码

```java
// 【修复】只有非超级管理员才过滤集团ID
if (!"000000".equals(currentTenantId)) {
    sql.append("AND u.tenant_id = ? ");
    params.add(currentTenantId);
}
```

---

## 📝 修改的方法

1. **list() 方法** - 列表查询
2. **export() 方法** - 导出功能

两个方法都使用相同的逻辑：
- 判断当前用户的 `tenant_id`
- 如果是 `000000`（超级管理员），则不添加集团过滤
- 如果不是 `000000`（集团管理员），则添加集团过滤
